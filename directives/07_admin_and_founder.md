## Admin Dashboard

**File:** `frontend/pages/admin-dashboard.html`
**JS:** `frontend/js/admin-dashboard.js`
**Login:** `admin@stemnestacademy.co.uk` / `StemNest2024!`

### Status: ✅ COMPLETE — API-driven

### What it does
- View all bookings (pending, scheduled, completed)
- Assign teachers to bookings
- Create new teacher accounts
- Manage courses
- View class reports

### Data Load
```js
_loadAdminFromAPI() → GET /api/sync/dashboard/admin
```
Returns: `bookings`, `classReports`, `pipelines`, `courses`, `tutors`, `sales`

Note: `materials: []` is returned as empty array — the `companion_materials` table does not exist in the DB.

### Create Teacher
```js
POST /api/users
{
  name, email, password, role: 'tutor',
  staff_id,    // auto-generated by nextTeacherId()
  subject, courses, gradeGroups, availability, dbs
}
```
Staff ID is auto-generated: `CT001`, `CT002`, etc. If taken, backend auto-increments.

### Assign Teacher to Booking
```js
PUT /api/bookings/:id/assign
{
  tutorId: "<UUID>",    // MUST be UUID, not staff_id
  classLink: "https://meet.google.com/...",
  date: "2026-05-20",
  time: "14:00"
}
```
After assign: booking moves from "Pending" to "Scheduled" tab.

### `nextTeacherId()` — How Staff IDs Are Generated
```js
function nextTeacherId(subject) {
  const prefix = SUBJECT_PREFIX[subject] || 'TT';
  // SUBJECT_PREFIX = { Coding: 'CT', Maths: 'MT', Sciences: 'ST' }
  const existing = window.ADMIN_DATA.tutors
    .map(t => t.staff_id || '')
    .filter(id => id.startsWith(prefix))
    .map(id => parseInt(id.slice(prefix.length)) || 0);
  const next = existing.length ? Math.max(...existing) + 1 : 1;
  return prefix + String(next).padStart(3, '0');
}
```

## Founder / Super Admin Dashboard

**File:** `frontend/pages/super-admin.html`
**JS:** `frontend/js/super-admin.js`
**Login:** `founder@stemnestacademy.co.uk` / `Founder2024!`

### Status: ✅ COMPLETE — API-driven

### What it does
- Full platform reporting: sales, classes, revenue, expenses
- Teacher management (active / discontinued)
- Student management
- Platform settings (seal, signature, birthday messages)

### Data Load
```js
_loadSAFromAPI() → GET /api/sync/dashboard/admin
```
Populates `window.SA_DATA = { bookings: [], tutors: [], sales: [] }`.

**Critical:** `getTeachers()`, `getBookings()`, `getSalesPersons()` all read from `window.SA_DATA` — NOT from localStorage. The old `sn_teachers` localStorage key is dead.

### Teachers Tab
- Shows all tutors from `window.SA_DATA.tutors`
- "Active" tab: `t.is_active !== false`
- "Discontinued" tab: `t.is_active === false`
- "Discontinue" button calls `DELETE /api/users/:id` (soft delete — sets `is_active = false`)

### Revenue / Expenses
- Revenue calculated from `payments` table
- Expenses stored in `localStorage.sn_expenses` (manual entry — no DB table yet)

## Pre-Sales Dashboard

**File:** `frontend/pages/presales-dashboard.html`
**JS:** `frontend/js/presales-dashboard.js`
**Login:** `presales@stemnestacademy.co.uk` / `StemNest2024!`

### Status: ✅ COMPLETE — API-driven

### Data Load
```js
_loadPresalesFromAPI()
  → GET /api/sync/dashboard/presales  (bookings)
  → GET /api/users?role=tutor          (teachers with UUID + staffId)
  → GET /api/users?role=sales          (sales persons with UUID + staffId)
```

### Teacher Dropdown
Teachers stored as `{ id: UUID, staffId: "CT004", name: "...", subject: "..." }`.
Dropdown value is always the **UUID** — never the staff_id.

### Schedule Demo
```js
PUT /api/bookings/:id/assign
{
  tutorId: "<UUID>",     // from dropdown value
  classLink: "...",
  date: "2026-05-20",
  time: "14:00",
  salesId: "<UUID>"      // optional
}
```

### Tabs
- **Incoming:** `status === 'pending'` bookings
- **Scheduled:** `status === 'scheduled'` bookings
- **Completed:** `status === 'completed'` demo bookings
- **Incomplete:** `status === 'incomplete'` bookings
- **Cancelled:** `status === 'cancelled'` bookings
